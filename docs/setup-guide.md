# /setup 사용 가이드

이 시스템을 처음 받았다면 가장 먼저 `/setup`을 실행해야 합니다. 5~30분 인터뷰를 통해 여러분 회사에 맞게 시스템이 자동 설정됩니다.

---

## 셋업 3단계

```
Phase 1 (필수, 5분)   →  /setup          # 회사 기본 정보
Phase 2 (권장, 10분)  →  /setup-tone     # 톤 학습
Phase 3 (선택, 15분)  →  /setup-domain   # 카테고리/벤치마크/이미지
```

각 Phase가 끝나면 즉시 시스템이 작동 가능한 상태가 됩니다. 다음 Phase는 원할 때 진행해도 됩니다.

---

## Phase 1 — `/setup` (5분)

**목적**: `knowledge/brand-facts.md` 파일을 자동으로 생성합니다. 이 파일이 있어야 `blog-writer`가 회사 정보를 알고 글을 씁니다.

`setup-interviewer` 서브에이전트가 다음 7개 질문을 한 번에 한 개씩 진행합니다:

| # | 질문 | 어디에 쓰이나 |
|:---|:---|:---|
| Q1 | 회사명 | 모든 글에 등장 |
| Q2 | 한 줄 소개 | 도입부 4줄 공식 "자격" 부분 |
| Q3 | 주력 카테고리 1~3개 | keyword-bank 생성 + 글 톤 |
| Q4 | **자랑할 만한 실수치 1~5개 (사실만)** | 권위(Authority) 핵심. 가장 중요. |
| Q5 | 인증/수상 | 권위 보강 |
| Q6 | 타겟 고객 한 줄 | 글의 시점/문체 결정 |
| Q7 | 사용 금지 단어 | banned-words.json 시드 |

**준비물**: 회사 정확한 수치(추정 금지). 모르면 그 자리에서 사장님께 한 번 확인하세요. 추측 숫자가 들어가면 신뢰가 박살납니다.

**완료 후**: `knowledge/brand-facts.md` 자동 생성. `/blog-new`로 즉시 첫 글 가능.

---

## Phase 2 — `/setup-tone` (10분)

**목적**: 여러분 회사 블로그 글을 자동 수집해서 AI가 톤을 학습하게 합니다. 첫 글의 품질이 크게 올라갑니다.

**진행 방식**:
1. 여러분 회사 블로그 URL 3~5개 입력 (운영 중인 글)
2. 회사 블로그가 없다면? 같은 산업에서 톤이 좋다고 생각하는 경쟁사/벤치마크 회사 URL 입력
3. `scripts/setup-tone-fetch.js`가 자동으로 본문 추출 → `knowledge/tone-samples/real-blog-posts.txt`에 저장
4. AI가 시그니처 표현 5개 자동 추출 → 사용자에게 검증 요청
5. 확정 시 `CLAUDE.md`에 "톤 시그니처" 섹션 자동 추가

**팁**:
- URL 3~5개에서 80KB 이상 수집되는 게 좋습니다 (글 5~10편 분량)
- 너무 적으면 톤 학습이 부정확해집니다
- 네이버 블로그 / 티스토리 / 워드프레스 / `<article>` 태그 시맨틱 HTML 모두 지원

**수동 모드**: URL 수집이 모두 실패하면 수동 모드로 전환됩니다. 글 5~10편을 직접 복사해서 `knowledge/tone-samples/real-blog-posts.txt`에 붙여넣으면 됩니다.

---

## Phase 3 — `/setup-domain` (15분)

**목적**: 카테고리별 키워드 뱅크 + 산업 금칙어 + 이미지 디자인 시스템 설정.

**진행 방식 (카테고리당 3질문)**:
1. **Q1**: 이 카테고리의 주요 키워드 5~10개?  → `keyword-bank/{slug}.yml` 생성
2. **Q2**: 이 카테고리에 적용되는 법령/규제 단어?  → `knowledge/banned-words.json`에 추가
3. **Q3**: 이 카테고리 업계 벤치마크 수치?  → `knowledge/conversion-benchmarks.md` 업데이트

**이미지 디자인 시스템 (선택, 5분)**:
- 브랜드 컬러 3개 hex (배경 / 본문 / 포인트)
- 폰트 스타일 (산세리프 / 세리프 / 혼합)
- 로고 텍스트 (이미지에 박힐 정확한 표기)
- → `.env`의 `BRAND_NAME`, `BRAND_BG_COLOR`, `BRAND_FG_COLOR`, `BRAND_ACCENT` 자동 채움

**의료/뷰티 카테고리 권장**: `medical-law-checker` 서브에이전트 활성화. 의료법 위반 한 번 걸리면 사업이 망합니다.

---

## 셋업 후 점검

```
/blog-new "테스트 키워드"
```

체크리스트:
- [ ] 회사명이 정확히 등장하는가
- [ ] 회사 실수치가 자연스럽게 인용되는가 (추측 숫자 없음)
- [ ] 톤이 여러분 회사 톤과 일치하는가
- [ ] 이미지에 정확한 브랜드명이 박혀있는가

문제가 있으면 `knowledge/brand-facts.md`를 직접 편집하세요. (gitignored이라 안전)

---

## 자주 묻는 질문

**Q. /setup을 다시 하면?**
이미 `brand-facts.md`가 있으면 덮어쓰기 확인을 묻습니다. YES 시 처음부터 다시 진행.

**Q. 일부만 답하고 끝낼 수 있나요?**
빈 답은 "없음"으로 저장됩니다. 나중에 `brand-facts.md`를 직접 편집해도 됩니다.

**Q. 회사가 여러 카테고리를 운영하면?**
주력 1~3개에 집중하세요. 4개 이상이면 톤이 분산되어 글 품질이 떨어집니다. 추가 카테고리는 별도 레포로 관리 권장.

**Q. /setup 실행 중 오류가 나면?**
`docs/troubleshooting.md` 참조.
