# 시스템 진화 기록 (How This Was Built)

이 문서는 이 블로그 자동화 시스템이 어떻게 진화했는지 시간순으로 정리한 기록입니다.
"왜 이런 구조가 됐는지"를 이해하고 싶을 때 참고하세요.

> 업센스가 실제로 블로그를 운영하며 다듬어 온 자동화 환경입니다.
> 처음부터 완벽하게 설계되지 않았습니다. 문제를 발견할 때마다 한 단계씩 보완했어요.

---

## 6단계 진화

### 0단계 — 스캐폴드
가장 먼저 한 일은 **외부 의존성 0의 빈 뼈대**를 만든 것입니다. `package.json` + 폴더 5개. `npm install` 한 번도 안 했어요. Node 20+ 내장 fetch만 사용합니다.

핵심 결정:
- 외부 npm 패키지 0 — 누구나 즉시 실행 가능
- 자동 발행 코드 의도적 미포함 — 사람이 검수해야 한다

### 1단계 — knowledge/ (단일 진실 공급원)
첫 글을 써봤더니 AI가 추측한 회사 수치가 들어갔어요. **신뢰가 박살날 위험.** 그래서 회사에 대한 모든 사실을 한 곳에 박았습니다. `knowledge/brand-facts.md`. AI는 이 파일에 있는 숫자만 사용하도록 강제.

이 한 파일이 2주 작업의 절반 효과를 만들었습니다. **단일 진실 공급원(Single Source of Truth)** 이 가장 큰 ROI.

### 2단계 — 자동화 + 훅 (결정론 검증)
검수를 매번 사람이 하면 안 합니다. 안 하니까 품질이 떨어져요. 그래서 `quality-check.js`(7항목 채점) + `duplicate-check.js`(6-gram Jaccard) + PostToolUse 훅을 만들었습니다. `post.md`를 저장하면 자동으로 채점됩니다.

핵심 원칙: **"잘 썼어?"를 LLM에 묻지 않는다. 결정론적 코드에 맡긴다.**

### 3단계 — 슬래시 + 서브에이전트 + 키워드뱅크
한 AI한테 다 시키니 부담이 컸어요. 책임 분리합니다. 슬래시 커맨드 4종(`/blog-new` 등) + 서브에이전트 4명(리서처/라이터/리뷰어/의료법 검사관) + 키워드 뱅크 yml 4종.

### 4단계 — 발행 어시스턴트 (`preview.js`)
글 다 만들었는데 발행이 제일 짜증났어요. 매번 30분 복붙. **죽고 싶었어요.** 그래서 self-contained HTML 어시스턴트를 만들었습니다. 좌측에 섹션별 카드 + 복사 버튼 2종, 우측에 메타카드 + 이미지 갤러리.

이거 만드는 데 3번 깨졌어요:
1. **1차**: UI 깨짐 (복사 버튼 검정 박스, 헤더 겹침, 표 오버플로) → CSS 정리
2. **2차**: 헤더 분리 (`flex-direction: column`)
3. **3차**: 클립보드 전쟁
   - 버그 1: `onclick`에 HTML 박았더니 따옴표 들어가서 JS 깨짐 → JSON에 박고 인덱스만 전달
   - 버그 2: `navigator.clipboard.write` API가 네이버 에디터 호환 X → **숨긴 contentEditable div + `document.execCommand('copy')`** (옛날 방식)
   - 추가: 네이버 H2 손실/표 차단 → 정규화 함수 6종 (h2 → 굵은 단락, table → 시각화 안내)

새벽 5시까지 디버깅했습니다. 진짜로요.

### 5단계 — 템플릿화 (이 공개판)
원래 이 시스템은 한 회사 전용이었어요. 외부 공개를 위해 회사 데이터를 전부 빼고 `/setup` 인터뷰로 누구나 자기 회사에 맞게 채울 수 있도록 템플릿화했습니다.

`knowledge/brand-facts.template.md` → `/setup` 5분 인터뷰 → `knowledge/brand-facts.md` 자동 생성.

## 5가지 빌드 교훈

1. **시작은 작게, 점진적으로 키우기** — 처음부터 완벽 설계 X. 0단계 → 글 한 편 → 문제 발견 → 보완.
2. **사용자가 알려준 정보가 가장 강력** — AI가 똑똑한 게 아닙니다. 사실을 가진 사람이 똑똑한 거예요.
3. **자동화는 사람이 검수해야 할 곳을 명확히 하는 것** — 발행은 사람이 합니다.
4. **단일 진실 공급원이 가장 큰 ROI** — `brand-facts.md` 한 파일이 2주 작업의 절반 효과.
5. **결정론적 검증 > LLM 검증** — "잘 썼어요"보다 "11회/0개/1.3%" 출력이 신뢰할 만함.
